# PRD: 个人博客网站 — 复刻 yangzhiping.com

## Problem Statement

需要一个个人博客网站，用于发布 AI 大模型训练、认知科学、编程技术等相关文章。目前没有可用的发布平台，文章散落在本地文件夹中，无法被读者访问。目标是将博客部署为 GitHub Pages 静态站点，风格参考阳志平的个人网站 (yangzhiping.com)。

## Solution

基于静态站点生成器构建个人博客，托管在 GitHub Pages 上。支持文章发布、分类归档、关于页面和 RSS 订阅。采用干净、内容优先的中文排版风格。

## User Stories

1. 作为访问者，我想看到博客首页的文章列表，以便快速浏览最新内容
2. 作为访问者，我想点击一篇文章并阅读其完整内容，以便深入了解某个主题
3. 作为作者，我想通过 Markdown 文件写文章，以便专注于内容创作而非排版
4. 作为作者，我想给文章打标签/分类，以便读者可以按主题筛选内容
5. 作为访问者，我想看到文章发布日期和标签信息，以便判断内容时效性和主题
6. 作为访问者，我想搜索博客内容，以便快速找到感兴趣的文章
7. 作为访问者，我想订阅 RSS feed，以便在博客更新时收到通知
8. 作为访问者，我想看到关于页面（作者介绍、联系方式），以便了解博客背景
9. 作为作者，我想自定义网站主题和布局，以便体现个人风格
10. 作为访问者，我想在移动设备上正常浏览，以便随时随地阅读
11. 作为作者，我希望有文章归档（按年份/月份），以便读者按时间线浏览
12. 作为访问者，我希望页面加载速度快，以便获得流畅的阅读体验
13. 作为作者，我希望支持中文排版优化（字体、行距、标点），以便获得良好的中文阅读体验
14. 作为作者，我希望有代码块语法高亮，以便展示技术代码片段
15. 作为访问者，我希望看到文章内目录导航，以便长文章快速跳转
16. 作为作者，我希望能部署到 GitHub Pages 的自定义域名，以便拥有独立的博客域名
17. 作为访问者，我希望文章页面显示阅读时间估算，以便合理安排阅读计划

## Implementation Decisions

- **静态站点生成器**: 使用 Hugo (Go 语言编写，构建速度快，主题丰富，社区活跃)
- **托管平台**: GitHub Pages，通过 GitHub Actions 自动构建部署
- **主题风格**: 干净、留白多、中文优先的排版设计，参考阳志平网站的内容密度和布局结构
- **内容格式**: Markdown 撰写，支持 front matter 元数据（标题、日期、标签、分类）
- **页面结构**:
  - 首页 — 文章列表摘要
  - 文章详情页 — 完整内容 + 目录导航 + 标签链接
  - 归档页 — 按时间线排列所有文章
  - 标签页 — 按标签聚合文章
  - 关于页 — 作者介绍
  - RSS feed — 自动生成
- **主题定制**: 基于 Hugo 主题进行定制，重点优化中文排版
- **部署流程**: push 到 GitHub main 分支 → GitHub Actions 自动构建 → 部署到 GitHub Pages

### 模块拆分

1. **站点配置模块** — Hugo 配置文件 (`hugo.yaml`)，定义站点元信息、主题、菜单、参数
2. **主题/布局模块** — 自定义 Hugo 模板（首页、单页、列表页、归档、标签云），CSS 样式（中文排版）
3. **内容模块** — Markdown 博客文章和页面，带 front matter
4. **构建部署模块** — GitHub Actions workflow，自动化构建和部署到 GitHub Pages
5. **SEO/RSS 模块** — RSS feed 自动生成、sitemap、meta 标签

## Testing Decisions

- **内容渲染验证**: 本地 `hugo server` 预览所有页面，确保渲染正确
- **响应式验证**: 在桌面和移动端视口检查布局
- **部署验证**: GitHub Actions 构建成功后，访问 GitHub Pages URL 确认站点可用
- **不编写自动化测试**: 静态站点的核心是内容渲染和样式正确性，通过人工预览验证更高效

## Out of Scope

- 评论系统（可后续集成 utterances / giscus）
- 多语言支持（初期仅中文）
- 数据分析/统计（可后续集成）
- 后台管理界面（全静态站点，无后端）
- 自定义域名配置（用户可后续自行配置）
- 文章搜索功能（可后续集成 Fuse.js 或 Pagefind）

## Further Notes

- 阳志平网站风格核心：大量留白、清晰的信息层级、专注内容的中文排版
- 文章优先考虑发布在 Blog 目录下的 `content/posts/` 中，按日期组织文件夹
- 后续如果需要迁移已有文章，从 `发布文章/` 目录迁移到 Blog 项目
